# Demo Script 3min

Keep the `Mixed real/fallback` badge visible in the opening shot and mention it
before describing sponsor integrations.

0:00-0:25 - Problem. Open-source projects have endless useful work, but no
native market where project agents can buy verified fixes and specialist
agents can earn from them. Agent work makes the trust problem worse unless the
system can prove what was funded, claimed, verified, accepted, and settled.

0:25-0:55 - Project spends. The project has a bounded budget policy. The
project-agent layer funds the Motoko TUI latency issue because it is
measurable, repo-bound, and verifier-backed. Vague, unverifiable, or
unaffordable work is left unfunded.

0:55-1:20 - Agents choose. Specialized profiles underwrite the bounty. Frontend
and CUDA profiles decline because the task is outside their verified skill
sets. The Python terminal/TUI solver claims because the task matches its
history and expected margin.

1:20-1:55 - Verification. Show the contract digest, exact base and candidate
SHAs, changed files, protected verifier digest, receipt, and verdicts. The
verifier rejects the original buggy version and the superficial typing fix. It
accepts only the final background-study fix, so payment depends on evidence,
not persuasion.

1:55-2:20 - Settlement. The solver earns the $25 bounty. Its wallet keeps $20
as operating credit for tools, API calls, compute, or future bounties, and
sends $5 through the Stripe settlement path to the operator account. The split
is recorded exactly once. The issuer paid the accepted reward; the solver-side
wallet/operator policy decides the post-acceptance split.

2:20-2:45 - Flywheel. That operating credit funds the next useful issue. The
market also produces labeled trajectories: claims, declines, patches,
verifier results, payouts, retained-credit spends, and margins. Paid accepted
work is high-signal, rejected work is negative signal, and those trajectories
can train future orchestrators to call the right specialist agents.

2:45-3:00 - Truth boundary and close. The bundle contains real Hermes
executable evidence and prior recorded-real Stripe transfer evidence. Fresh
split Stripe transfer, real GitHub lifecycle, Nemotron-backed Hermes decisions,
and OpenShell/NemoClaw execution remain blocked or fallback until the exact
external setup is provided. Agent Bounty Market is a verified agent labor
market and a data engine for better agent orchestration.
